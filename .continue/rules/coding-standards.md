---
name: Coding Standards
description: Coding Standards
---

- Use JavaScript, not TypeScript.
- Match the existing folder structure.
- Never use inline styles.
- Never use !important in CSS.
- Always reuse existing global CSS before creating new styles.
- Component‑specific CSS must live in a file matching the component name (e.g., Component.css).
- Component‑specific class names must begin with the component name (e.g., .Component-button).
